# API
- 예약 가능 날짜/좌석 API(콘서트 조회)
- 좌석 예약 요청 API
- 포인트 충전/조회 API
- 결제 API
- 유저 대기열 토큰 발급 API

# 사용자 시나리오
- 예상 시나리오
    1. 고객 로그인 후 특정 콘서트 선택
    2. 예약 가능 날짜/좌석 목록 조회
    3. 예약 가능한 날짜/좌석 선택
    4. 좌석 예약 요청
        - 좌석 예약 요청 시 해당 고객에게 5분간 임시 배정
        - 배정 시간 내에 결제 미완료 시 임시 배정 해제
    5. 포인트 충전/조회
    6. 결제
## 기술
- Java 21, Spring Boot3, JPA, JWT
- JUnit5
